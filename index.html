Utility

Utility anathomy

[media:]subject[-value]

[media:]whatItDoesOrMean[-value]

`.whatItDoes` `.whatItDoes-value` `.media:whatItDoes-value`
  • Media and value are optional
  • All the parts of a utility are in camel case

  • Value part examples -major -3xl -extraBold -100 -180deg -38parrots

Block + Element + Modifier

.Block .Block__element .Block--modifier
  • .SomeBlock - блок (TitleCase)
  • .SomeBlock__innerBlock — элемент
  • Модификаторы:
    • .SomeBlock--modifier — модификатор (camelCase)
    • .is-* — состояние блока (is-selected)
    • .has-* — наличие потомка, или состояние (has-focus например)
  • .do-* — префикс классов для навешивания скриптов

				### Reserved prefixed modifier

				`.do-` the hook for using in scripts

				`.is-` component state modifier

				`.has-` state or present of a descendant

				`.G-` global context class

				Don't worry, it's okay that prefixed modifiers looks like a utilities.
				The main thing is that they have meaningful names. `do-blockExpand` for example.

				(you can use more)

				+ components mixing
