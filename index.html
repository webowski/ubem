Yes, yet another CSS naming system. Here is nothing new, just a useful set of known practices.

UBEM is based on using two entities - utility and component. The main goal is to use most recognizable combination of letter cases and parts separators.

Utility

Utility anatomy

<div class="media:subject-value"></div>
  • Media part is optional and used for responsive styles
  • Value part is optional and used for utility varied values
  • All the parts of a utility are in camel case

Component

Anatomy

<article class="SomeBlock SomeBlock--modifier do-scriptMagic is-stateModifier">
	<header class="SomeBlock__element">

	</header>
	<div class="SomeBlock__anotherElement">

	</div>
</article>
  • .SomeBlock — block name in PascalCase
  • .SomeBlock__innerBlock — element in camelCase
  • .SomeBlock--modifier — modifier in camelCase

Prefixed modifiers

  • .is-* — component state modifier (is-selected)
  • .has-* — state or presence of a descendant (has-error)
  • .do-* — the hook for using in scripts
  • Global context modifier with G prefix. .G-* (.G-logged, .G-admin). If necessary you can add global context classes to the body tag.

Don't worry, it's okay that prefixed modifiers looks like a utilities. The main thing is that they have meaningful names. do-blockExpand for example.

Avoid

Mixing components <div class="SomeBlock AnotherBlock"></div>
