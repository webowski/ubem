Yes, the yet another CSS naming system. Here is nothing new, just a useful set of known practices.

UBEM stands for Utility and Block-Element-Modifier.

At first, the example how it might look:

Utility

Utility anatomy

[media:]subject[-value]

[mediaPoint:]whatItDoesOrMean[-specificValue]

  • Media and value are optional
  • All the parts of a utility are in camel case

Utility examples

  • Media part examples sm: md: lg: xl: mdDown: xsOnly: print: iron:
  • Value part examples -major -3xl -extraBold -100 -180deg -38parrots
				- margin, padding, width
					- sm:py-2 sm:py-sm
				- text
				- grid
				- custom
				md:jc-between
				xsOnly:d-none
				print:d-none
				sm:gap-sm
				sm:gap-3xl
  • Сетка .row, .col-*, .grid, .gutter-*
  • Утилиты
    • Кастомные .whatItDoesOrMeans (.expandLinkArea, .visuallyHidden)
    • Атомарные
      • блочные: d, .justify-*, .align-*
      • поля и отступы .p-* .mx-*
      • текстовые: .text-*, .font-*, .color-*, .list-* (.fontSize-sm)

Exception

Global context utility with G prefix. .G-* (.G-logged, .G-admin). If necessary you can add global context classes to the body tag.

Block-Element-Modifier

Anatomy

.Block .Block__element .Block--modifier

Modifiers

  • .SomeBlock - блок (TitleCase)
  • .SomeBlock__innerBlock — элемент
  • Модификаторы:
    • .SomeBlock--modifier — модификатор (camelCase)
    • .is-* — состояние блока (is-selected)
    • .has-* — наличие потомка, или состояние (has-error например)
  • .do-* — префикс классов для навешивания скриптов

				### Reserved prefixed modifier

				`.do-` the hook for using in scripts

				`.is-` component state modifier

				`.has-` state or present of a descendant

				`.G-` global context class

				Don't worry, it's okay that prefixed modifiers looks like a utilities.
				The main thing is that they have meaningful names. `do-blockExpand` for example.

				(you can use more)

				### Denied

				+ components mixing
				~~`<div class="OneBlock AnotherBlock">`~~
